### TASK-009: Миграция на App Router и реорганизация структуры проекта

**Статус:** Завершено
**Приоритет:** Высокий
**Описание:** Миграция проекта с Pages Router на App Router и оптимизация структуры проекта согласно лучшим практикам Next.js.

#### План реорганизации:

1. **Создание базовой структуры:**

   - Сохранить `/app` для App Router
   - Сохранить `/components` для UI компонентов
   - Сохранить `/lib` для утилит и хуков
   - Сохранить `/public` для статических ресурсов

2. **Миграция с Pages Router на App Router:**

   - Создать маршруты:
     - `/app` (корень) - Основной макет и главная страница
     - `/app/auth` - Страница аутентификации
     - `/app/settings` - Страница настроек
     - `/app/project` - Страница проекта
     - `/app/create-project` - Страница создания проекта

3. **Организация компонентов:**

   - Сохранить `/components/ui` для переиспользуемых UI компонентов
   - Сохранить `/components/common` для общих компонентов макета
   - Создать каталоги для компонентов по функциональности:
     - `/components/auth` - Компоненты аутентификации
     - `/components/project` - Компоненты проектов
     - `/components/settings` - Компоненты настроек
     - `/components/project-creation` - Компоненты создания проекта

4. **Стратегия рефакторинга:**
   - Поэтапная миграция:
     1. Рефакторинг существующих страниц App Router для прямого включения кода из Pages Router
     2. Удаление файлов Pages Router после миграции всей функциональности
     3. Проверка сохранения всей функциональности после каждой миграции
   - Клиентские vs Серверные компоненты:
     - Добавление директивы `"use client"` для компонентов, требующих клиентских функций
     - Использование серверных компонентов для статических частей для повышения производительности
