## Сводка

- **Горизонтального скролла страниц нет ни на одной ширине** (DOM-проверка чистая):
  «наезды» — это сжатие внутри flex-рядов и обрезка грид-колонок, не переполнение страницы.
- Консоль/сеть чистые на всех турах (кроме намеренной 404-пробы).
- Тёмная тема специфичных мобильных проблем не добавляет — все находки общие.

## Системное (блок A)

- [ ] **P0 safe-area отсутствует**: нет `viewport-fit=cover`, ни одного `env(safe-area-inset-*)`.
  На iPhone композер чата, нижние кнопки шитов и drawer ложатся под home-индикатор,
  топ-бары — под чёлку в landscape.
- [ ] **P1 тач-цели меньше 44px** (замер на 390, высота фактическая):
  - 12px — карандаш/корзина модуля (course edit); 14px — «+ Новый таб» (drawer);
  - 16px — карандаш/корзина тарифа, «Удалить карту»; 17px — «В закладки», «+» категорий drawer;
  - 20px — лайк/комментарии поста, радио «Выбрана для выплат»; 22px — send чата;
  - 24px — «···» канала (drawer); 28px — «···» сообщения, SegmentedControl, фильтры транзакций;
  - 32px — пиллы настроек (уйдут в блоке B), «Выйти», крестик модалки;
  - 34px — строки каналов drawer; 36px — бургер, строки уроков, «Опубликовать».
  Правило: визуальный размер можно оставить — расширять тап-область (padding, hit-slop).
- [ ] **P1 стек хрома на мобиле**: топ-бар (48) + тайтл-бар (48) + [сегмент-контрол (48)] +
  [«К списку уроков» (48)] = до 190px служебных полос на экране 844px (курс — худший).
  Схлопнуть: сегмент-контрол — в строку тайтл-бара; «К списку уроков» — компактнее (блок D).
- [ ] **P2 конвенция полей**: разнобой px-3/px-4 по страницам → правило: контент ≥16px от кромки
  на <md, 24px на md+ (фикс по ходу блоков).
