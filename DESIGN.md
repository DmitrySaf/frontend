# Bean — DESIGN.md

> Снято с живого кода (app/globals.css + src/shared/components). Актуально на этап 17 (дизайн-ревью, в процессе): включает тёмную тему, самохостинг Onest и удаление `bean-design-system`.
> Источник истины — сам код; этот файл — карта для агентов и ревью.

## 1. Visual Theme

Светлая продуктовая тема Apple/Whop-школы: белые поверхности, тонкие серые рамки-волоски, крупные радиусы, один синий акцент. Плотность — комфортная продуктовая (не маркетинговый воздух и не кокпит). Светлая тема — база; тёмная реализована ретрофитом через переворот токенов (см. §2).

## 2. Color

Токены объявлены в `app/globals.css` (@theme → CSS-переменные, HSL-триплеты). Шкалы тональные, выверены по извлечённым токенам apple.com/whop.com.

| Роль | Токен | Значение | Использование |
|---|---|---|---|
| Фон приложения | `--color-background` | белый | страницы, карточки |
| Каркас/рельсы | `gray-100` | hsl(0 0% 96%) | фон рейла/drawer, треки, ховеры ghost |
| Рамки | `gray-200` | hsl(0 0% 90%) | борта карточек, инпутов, разделители |
| Активные строки | `gray-200/70` + ring `gray-300/70` | — | выбранный канал/таб/урок |
| Вторичный текст | `gray-500` / `gray-600` | 55% / 45% | метаданные / подписи (AA на белом) |
| Поверхность | `--color-surface` | white / dark 0.15 | карточки/бары/поля (`bg-surface`, переворачивается в тёмной) |
| Основной текст | `--color-ink` | oklch(0.193) / dark 0.96 | body (`text-ink`, переворачивается в тёмной) |
| Акцент | `primary-500` | hsl(210 100% 45%) ≈ #0074E6 | кнопки, ссылки-действия, фокус, селекция |
| Акцент-ховер | `primary-400` | 49% | hover primary-кнопки (светлеет, как apple.com) |
| Акцент-нажатие | `primary-600` | 40% | active; также текст-ссылки на белом |
| Светлый тинт | `primary-50` | 97% | фон выбранной опции (тип таба, тариф) |
| Заливка | `--color-fill` / `-hover` | 0.962 / dark 0.26 | серая кнопка «Отмена», кружок закрытия (Apple systemFill) |
| Скрим | `--color-scrim` | black/18% / dark 60% | подложка модалки — светлая, работу делает blur |
| Опасность | `--color-danger` | oklch(0.59 0.196 36) | удаление, выход, ошибки |
| Бренд-лайм | только логотип и пилл «Закреплено» (#DEFCAD) | — | не системный токен |

Правила: один акцент; цвет = состояние или действие, не украшение; серый текст на цветном фоне запрещён (использовать тёмный той же гаммы/прозрачность белого); графики — primary-500 для текущего/линий, gray-100+рамка gray-300/70 для фоновых баров.


## 3. Typography

- Семейства: `--font-sans` = **Onest**, самохостинг через `@font-face` в [globals.css](app/globals.css) (`public/fonts/onest-{latin,latin-ext,cyrillic}.woff2`, variable — один файл на веса 100–900) — основной UI/боди-шрифт с первоклассной кириллицей. `next/font/google` не используется: сеть окружения флапала и шрифт не скачивался, а `@theme` плодил хрупкую индирекцию `var(--font-sans)→var(--font-onest)` — семейство подключено прямым именем.
- `--font-mono` = **системный моно-стек** (`ui-monospace`, SF Mono, Menlo, Consolas) — только цифры/деньги/тайминги. Geist Mono убран (лишний вес ради латиницы, которой в цифрах не видно). `<html lang="ru">`.
- База: 16px / line-height 1.5; `--text-sm` 14px/1.43 — списки, вторичные строки; заголовки страниц 24px bold, секций 15–17px bold, модалок `text-2xl` + `tracking-tight`.
- Моно — только цифры/деньги/тайминги/OTP (`font-mono`): статистика, `мм:сс`, счётчики.
- Микротекст 11–13px используется плотно (метки каналов, подписи) — контраст держать ≥4.5:1 (gray-500+).
- Копирайтинг: русский, sentence case, на «вы», без эмодзи; деньги «₽ 990 / мес» с неразрывными пробелами.

### 4.1 Закрытие модалок

Раскладка кнопок и наличие крестика выводятся из трёх осей: **что теряется** при случайном закрытии, **равноценны ли** варианты выбора, и **достижимость** (на телефоне верхний правый угол — худшая точка для пальца, низ — лучшая).

**Каталог раскладок.** Полный список известных паттернов — включая те, что мы сейчас не применяем: если понадобится новый тип окна, выбирать из этого списка, а не изобретать.

| | Раскладка | Референс | Статус у нас |
|---|---|---|---|
| **P1** | Пара кнопок справа внизу, крестика нет | macOS-диалоги, Material 3 basic dialog | подтверждения (десктоп) |
| **P2** | Пара справа внизу + крестик | Stripe, GitHub Primer, Shopify Polaris | формы (десктоп) |
| **P3** | 50/50 на всю ширину нижней панели | iOS-алерты, Fluent ContentDialog, Vercel | подтверждения (мобильный), `layout="split"` |
| **P4** | Одна кнопка на всю ширину + крестик, «Отмены» нет | оплата/логин/пейволы у Apple в вебе, Whop | `CheckoutModal` |
| **P5** | Стопка на всю ширину: действие сверху, отход призраком под ним | мобильные шиты, Material full-screen | формы (мобильный), `layout="stack"` |
| **P6** | Верхняя панель: «Отмена»/✕ слева, действие текстом справа, футера нет | iOS page sheet, Material full-screen dialog | **не используется** — кандидат для полноэкранных мобильных форм |
| **P7** | Только крестик, кнопок действий нет | информационные окна | `InviteDialog`, `AuthRequiredDialog` |

Отдельно: **свайп вниз** — не раскладка кнопок, а замена клику снаружи на мобильном. Именно им Apple компенсирует отсутствие крестика в шите. **Не реализован**; нужен, если модалки на `<768px` будут переведены в bottom sheet.

Если крестик на мобильном всё же нужен (P6) — он ставится **слева**: правый верхний угол недостижим большим пальцем на крупном экране.

| Тип окна | Десктоп | Мобильный | Крестик | Клик снаружи | Кто |
|---|---|---|---|---|---|
| Подтверждение (необратимое) | пара справа | 50/50 (`layout="split"`) | нет | **нет** | `DeleteDialog` |
| Форма / создание | крестик + пара справа | стопка (`layout="stack"`) | да | только если `!isDirty` | `Community/Channel/Tier/AddCard` |
| Одно действие (конверсия) | крестик + кнопка на всю ширину | то же | да | да | `CheckoutModal` |
| Информационное | только крестик | то же | да | да | `InviteDialog`, `AuthRequiredDialog` |

Правила, из которых это следует:

- **Один подписанный выход на окно.** Есть футер с «Отменой» → крестика нет (иначе неподписанный контрол дублирует подписанный). Нет футера → крестик и есть выход.
- **50/50 — только для симметричного выбора.** В форме «Отмена» и «Создать» не равноценны: одна — цель, вторая — аварийный выход. Дать им по половине ширины — соврать про вес. Форма на мобильном = стопка: действие сверху, отход под ним.
- **В подтверждении акцент (плотная заливка) — у безопасного варианта.** Деструктив — `destructiveTonal`, а не сплошной `destructive`: залитый красным блок на пол-экрана превращает необратимое действие в самую крупную мишень окна. Красный обязан читаться как опасность, а не как главный акцент (логика iOS-алерта).
- **Клик снаружи и Esc — ускорители, а не выход.** Разрешены ровно там, где закрытие ничего не стоит. Крестик в подтверждении опасен не тем, что можно случайно удалить (случайное закрытие как раз безопасно), а тем, что даёт закрыть окно **не прочитав вопрос**.
- **Футер липкий.** Иначе на высокой модалке «Отмена» уезжает за скролл, и единственным выходом остаётся крестик в самом недостижимом углу. Контент растворяется под футером градиентом (`::before`), а не обрывается по жёсткой линии.

Паттерны: селекция опций = рамка `primary-500` + фон `primary-50` (радио/плитки/тарифы); строки списков (каналы/уроки/табы) = r10, hover `gray-200/45`, active `gray-200/70` + inset-ring `gray-300/70`; «···»-меню появляются по hover на десктопе и видимы на таче.

## 5. Layout

- Каркас: rail 60px (gray-100) → сайдбар сообщества 256px (белый, борт gray-200) → контент; `<768px` — Discord-drawer (85vw, max 340) + топ-бар с бургером.
- Радиусная система: строки/малые контролы 10px · контролы/меню 12px (`rounded-xl`) · поля/плитки 14px (`rounded-lg`) · карточки 16px (`rounded-2xl`) · модалки/логин 24px (`rounded-3xl`) · пиллы full. Произвольные значения вне шкалы не вводить.
- Сетка 4px; карточки `p-5`; контентные колонки: чат full-width, посты 720px, урок 768px (`max-w-3xl`), настройки `max-w-2xl`, витрина 1000px.
- Тени: карточки `shadow-sm`, поповеры `shadow-lg`; глубина в основном рамками и слоями серого. Модалки — `shadow-modal`, слоистая (hairline-обводка + контактная + две амбиентные): одной тени не хватает — широкая размывается в грязь, короткая липнет к краю. В тёмной теме тень почти не читается, поэтому глубину держат приподнятая поверхность, hairline-обводка и внутренний блик сверху (`inset 0 1px 0 white/6%` — панели macOS ловят свет краем).
- z-слои: контент → dropdown/tooltip/modal = `z-50` (радикс-порталы) → drawer `z-50`; девятисотых значений нет.

## 6. Motion

Токены в globals: `--ease-out-quart` (базовый выход/ховеры), `--ease-out-expo` (входы слоёв), `--ease-in-out-quart` (движение на экране), `--ease-drawer` (iOS-кривая). Длительности: пресс 100–160 · тултипы 125–150 · меню 150–200 · модалки 240/160 (выход быстрее входа) · контент ≤300; всё ≤300мс.

- Слои входят от триггера (origin-aware) с масштабом от ~0.95, никогда от 0. **Исключение — модалка**: она ни к чему не привязана, поэтому зумится из центра экрана.
- Модалку центрировать **только флексом на обёртке**, не через `translate(-50%,-50%)` на самой карточке: `tailwindcss-animate` подменяет `transform` целиком, и на время анимации коррекция обнуляется — карточка вылетает из угла. Компенсировать это через `slide-in-from-*-1/2` нельзя: дробные утилиты у плагина не генерируются (в бандле их нет, ошибки при этом тоже нет — стиль просто молча не применяется).
- Появление нового контента (сообщения/комментарии/посты) — `@starting-style` через `[data-live]`: история не мигает, вставки всплывают (+6px, 240мс).
- Спринги (`motion`) только там, где есть жест/физика: drawer (bounce 0, dur 0.35, свайп-закрытие), пилюля сегмент-контрола, лайк-«пружинка» (CSS keyframes 350мс).
- Чарты: бары растут с каскадом 28мс, линия прорисовывается (`pathLength`), стат-карты — stagger 60мс.
- `prefers-reduced-motion: reduce` глобально гасит движение (transition/animation ≈0); в motion-компонентах `useReducedMotion` переключает на короткий фейд.
- Не анимировать: клавиатурные/сверхчастые действия, layout-свойства (кроме прогресс-баров и grid-rows схлопывания категорий).

## 7. Do / Don't

**Do**: держать один синий акцент; тонкие рамки вместо теней; radius только из шкалы; skeleton вместо спиннера на составных экранах; пустые состояния с иконкой-плиткой, заголовком и действием; тач-цели ≥44px.

**Don't**: `grid` без явных колонок в контейнерах с ограниченной шириной (неявный трек = `auto` раздувается под min-content и распирает контейнер вбок — в модалках всегда `grid-cols-1`, т.е. `minmax(0,1fr)`); цветные side-stripe борта; градиентный текст; стеклянные карточки без причины; зебра в таблицах; `transition: all`; scale(0)-появления; ease-in на UI; произвольные hex вне токенов (лайм логотипа — исключение); эмодзи как иконки (только lucide).
